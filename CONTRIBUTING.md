# Contributing to Label Alchemy

Thank you for helping make Label Alchemy better! This repository does **not** contain the extension's source code, so contributions here take the form of **great bug reports, feature ideas, and community answers** rather than pull requests against the product.

## Ways to contribute

- **Report a bug** with clear, reproducible steps → [Bug report](../../issues/new?template=bug_report.yml)
- **Propose a feature** and let others upvote it → [Discussions → Ideas](../../discussions/categories/ideas)
- **Answer questions** from other users → [Discussions → Q&A](../../discussions/categories/q-a)
- **Share how you use it** → [Discussions → Show & tell](../../discussions/categories/show-and-tell)

## How to file a bug that gets fixed fast

The [bug report form](../../issues/new?template=bug_report.yml) asks for everything below — please fill it in completely.

1. **Search first.** Check [existing issues](../../issues?q=is%3Aissue) and the [FAQ](https://docs.labelalchemy.dev) for a duplicate.
2. **One bug per issue.** Separate problems are easier to track and fix.
3. **Steps to reproduce.** Numbered, minimal, and deterministic — what you did, in order.
4. **Expected vs. actual.** What you thought would happen, and what actually happened.
5. **Environment.** Label Alchemy version, VS Code version, OS, and the Salesforce file type involved (Apex / LWC / Aura).
6. **Naming mode.** Deterministic, or AI naming (and which provider/model) — many issues are mode-specific.
7. **A minimal, redacted snippet** that triggers it, when relevant.
8. **Logs.** Any error text from the Output panel or a notification.

## ⚠️ Never include secrets

Do **not** paste API keys, license keys, org credentials, or proprietary source into a public issue or discussion. Redact identifiers and use a small synthetic example instead. For anything sensitive, email **support@labelalchemy.dev**.

## Code of conduct

By participating you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Be kind and constructive.
